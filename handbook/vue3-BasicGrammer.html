# 3.vue3核心语法

# 3.1 【OptionsAPI与CompositionAPI】

  • OptionsAPI，选项式、配置式

Options类型的API,数据、方法、计算属性等，是分散在：data、methods、computed中的，若想新增或者修改一个需求，就需要分别修改：data、methods、computed,不便于维护和复用。

  • CompositionAPI，组合式API

CompositionAPI可以用函数的方式，更加优雅的组织代码，让相关功能的代码更加有序的组织在一起

# 3.2 从setup开始

vue3的template里面，可以有多个根标签，但是vue2里面只能有一个根标签。

setup不包裹得vue语法，不需要用return返回所有声明的变量，因此我们倾向于写setup在script标签内，就不用最后写一个return了

setup中，函数的this指向windows，因此我们基本不用this

setup与传统的配置项能不能混合写，如果可以，以谁为准：

当vue2与vue3语法共存时，旧语法可以读取新语法中的内容，但是vue3语法setup语法块里面不能读取旧语法

当然了，name只能放在vue2标签里面，这一点令人

<script>

        name:'Person',
        setup(){

                console.log(666)
            }
            //一定别忘了return
            return {name,changeName}

# 3.4 使用reactive创建对象类型数据

const carObj = reactive({name:'Benz',price:2000})

reactive 和 ref 的区别

  1. ref支持基础类型，而reactive只支持对象。

  2. reactive不用 .value 进行赋值

  3. 当reactive重新分配一个对象，他就会失去响应式（但是单单修改某个属性可以）

    let car = reactive({brand:'Benz',price:100})
function changeCar(){
	car.price = 114 //改单个属性是可以的
	car = {brand:'Auto',price:5}
	car = reactive({brand:'Auto',price:5})
	//以上这两种写法都会失去响应式，因为堆内存指向的东西不一样了
    
	////引入一个函数Object.assign(obj1,obj2,obj3)//将obj2里面所有的键值对赋值到obj1里面，将obj3里面的每一个键值对加到obj1里【如果有多个参数的话】
    //最佳方案
    Object.assign(car,{brand:'Auto',price:5})
}

  4. ref只要用 .value 修改值，就一定是响应式的（对象也一样）

    • 基本类型选 ref
    • 响应式对象，层级不深，ref / reactive都可以
    • 如果层级很深，例如表单/表格，就用reactive（因为不用写.value）

# 辨析（v-show 和 v-if的区别）

  • v-if 是“真实的” 按条件渲染， 条件区块内的子组件会被销毁和重建，他也是惰性的，条件不符合时不会渲染

  • v-show 不论条件如何，始终会被渲染，它的显示与否，从根本上是由css中display: block/none来判断显隐的

  • v-show有较高的初始渲染开销（不论对错，一律渲染）， v-if有较高的切换开销（每次切换，都会重新渲染）

# :key的功能和作用

  • vue按照“就地更新”的策略，如果列表项已经渲染过了， 那么可以给每一个项一个唯一标识（：key）,这样就不用重新渲染了， 重新排序即可
  • 建议不要用 index 作为:key， 因为他不是每个元素的唯一标识
<template>
    <h2>v-for 之 :key 的用法</h2>
    <div v-for="(item,index) of carList" :key="index">{{item}}</div>
</template>

<script>
export default{
    data(){
        return{
            carList:["BMW","Honda","Mini","Benz"],
        }
    }
}
</script>

# 计算属性computed

虽然在前面讲到，我们可以在模板语法中直接使用表达式，但是那样的可读性很差，并且每次的判断结果不容易复用。

因此，vue推荐使用计算属性computed来描述依赖响应式状态的复杂逻辑。要注意以下几点

  • 响应式依赖（即我们在data中声明的对象）
  • 当computed使用响应式依赖对象，作为原变量进行计算时，值会实时、同步刷新
  • 但是如果不是响应式对象，那么不会同步刷新

示例如下

<template>
    <h2>计算属性</h2>
    <p>但是如你所见，这个方法太复杂了</p>
    <p>>=2吗？   {{testList.length>2 ? 'Yes' : 'No'}}</p>
    <p>computed计算属性可以传递一个处理好的变量</p>
    <p>{{lengthBigger2}}</p>

</template>

<script>
export default {
    data(){
        return{
                testList:["1000","600","840"]
        }
    },
    computed:{
        lengthBigger2(){
            return this.testList.length>2 ? 'Yes' : 'No'
        }
    }
}
</script>

# 侦听器watch

我们可以使用 watch 选项在每次响应式属性发生变化时触发一个函数。

示例：

<template>
    <h1>侦听器</h1>
    <h3>{{message}}</h3>
    <button @click="change">+1</button>
</template>

<script>
import { watch } from 'vue'
export default{
    data(){
        return{
            message:0
        }
    },
    methods:{
        change(){
            this.message++
        }
    },
    watch:{
        //newValue改变后的数据,oldValue改变前的数据
        message(newValue,oldValue){
            //每当数据发生改变，自动触发的函数
            console.log("之前是：",newValue,"\t现在是：",oldValue)
        }
    }
}
</script>
  • 侦听器的高级用法，当事件发生一定时间后触发侦听
let timer;
watch(searchName, (newValue, oldValue) => {
    // 使用定时器等待用户停止输入
    clearTimeout(timer);
    timer = setTimeout(() => {
        // alert(newValue)
    }, 500); // 在500毫秒内没有输入则触发
});

# ref

虽然vue已经为我们抽象了大部分对DOM的直接操作，但是如果需要直接访问DOM元素的话，我们需要使用特殊的 ref 方法进行变量绑定

挂载结束后，引用都会暴露在this.refs.变量名 之上

  • ref绑定的值不是响应式对象，因此无需声明变量，
  • ref元素最好不要重复，否则出大问题
  • ref元素是对DOM元素直接进行操作的方法，如果有其它方式的话，尽可能少用

下面这个例子给出了 ： 输出DOM节点、操作DOM元素、获取DOM内容 的样例

<template>
    <div ref="divRef" class="divDemo">{{content}}</div>
    <button @click="getElementHandel()">获取元素</button>
    <div ref="divChangeRef">empty</div>
    <input type="number" ref="inputName">
</template>

<script>
export default {
    data(){
        return{
            content:"Content Here"
        }
    },
    methods:{
        getElementHandel(){
            //输出ref对应的DOM元素
            console.log("获取元素-DOM is : ",this.$refs.divRef)
            //后面可以跟随原生js的语法，操作DOM
            console.log("修改元素内容",this.$refs.divChangeRef.innerHTML="WOOOOOOW!")
            //可以打印出元素里面的值
            console.log("input里面的值是：",this.$refs.inputName.value)
        }
    }
}
</script>
