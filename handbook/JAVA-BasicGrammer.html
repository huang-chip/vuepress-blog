# 哈希表hashmap

import java.util.HashMap;

public class HashMapExample {
    public static void main(String[] args) {
        // 创建一个HashMap,后面用于指定键和值的类型 <键类型, 值类型>
        HashMap<String, Integer> hashMap = new HashMap<>();

        // 1. put(key, value) - 将键值对添加到哈希表
        hashMap.put("Alice", 25);
        hashMap.put("Bob", 30);
        hashMap.put("Carol", 28);

        // 2. get(key) - 获取指定键对应的值
        int age = hashMap.get("Alice");
        System.out.println("Alice's age: " + age);

        // 3. containsKey(key) - 检查哈希表是否包含指定键
        boolean containsBob = hashMap.containsKey("Bob");
        System.out.println("Contains Bob: " + containsBob);

        // 4. remove(key) - 移除指定键及其对应的值
        hashMap.remove("Bob");
        System.out.println("After removing Bob: " + hashMap);

        // 5. size() - 获取哈希表中键值对的数量
        int size = hashMap.size();
        System.out.println("Size of HashMap: " + size);

        // 6. keySet() - 获取所有键的集合
        System.out.println("Keys in HashMap: " + hashMap.keySet());

        // 7. values() - 获取所有值的集合
        System.out.println("Values in HashMap: " + hashMap.values());

        // 8. isEmpty() - 检查哈希表是否为空
        boolean isEmpty = hashMap.isEmpty();
        System.out.println("Is HashMap empty? " + isEmpty);
    }
}

# switch多重选择

switch语句根据switch (表达式)计算的结果，跳转到匹配的case结果，然后继续执行后续语句，直到遇到break结束执行。当没有匹配到任何case时，执行default：

  1. switch还可以对比string类型，用 case "james": 即可
  2. 不要忘记写break以及default
public class Main {
    public static void main(String[] args) {
        int option = 99;
        switch (option) {
        case 1:
            System.out.println("Selected 1");
            break;
        case 2:
            System.out.println("Selected 2");
            break;
        default:
            System.out.println("Not selected");
            break;
        }
    }
}

# while循环

while循环在每次循环开始前，首先判断条件是否成立。如果计算结果为true，就把循环体内的语句执行一遍，如果计算结果为false，那就直接跳到while循环的末尾。

while (条件表达式) {
    循环语句
}

do while循环在每次循环结束后，判断条件，如果为false则退出循环，因此，do while会执行至少一次。

	do {
        sum = sum + n;
        n ++;
    } while (n <= 100);

    在OOP的术语中，我们把Person称为超类（super class），父类（parent class），基类（base class），把Student称为子类（subclass），扩展类（extended class）。

  • super方法以及子类继承

    由于在子类方法中，任何class的构造方法，第一行语句必须是调用父类的构造方法。如果没有明确地调用父类的构造方法，编译器会帮我们自动加一句super();，在父类没有空继承方法时，就会报错

    因此，在子类使用构造函数时，必须使用super方法

    在Java中，任何class的构造方法，第一行语句必须是调用父类的构造方法。

    class Student extends Person {
    protected int score;

    public Student(String name, int age, int score) {
        //如果不写super，那么Java会自动执行一条super();。自动调用父类的构造方法
        super(name, age); // 调用父类的构造方法Person(String, int)
        this.score = score;
    }
}

    • 组织继承

      sealed修饰符用于限制某个类的子类，以控制哪些类可以继承该类。

      final关键字

      • 当应用于类时，表示该类是最终的，不能被继承。
      • 当应用于方法时，表示方法是最终的，不能被子类重写。
      • 当应用于变量（成员变量或局部变量）时，表示变量是不可变的，即它的值不能被修改。

      non-sealed关键字：在sealed类的上下文中，可以使用 关键字来表示一个子类是非受限制的，即可以被其他类继承。

   }

这一点和python中的浅复制的赋值方案有很大不同，是将新变量新赋值了一个空间，而不是共用一个地址

public int search(int[] nums, int target) {
        int left = 0, right = nums.length - 1;
        while (left <= right) {
            int mid = (right - left) / 2 + left;
            int num = nums[mid];
            if (num == target) {
                return mid;
            } else if (num > target) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return -1;
    }

完成「力扣」第 374 题：猜数字大小；

当循环体只剩下两个元素的时候，一旦进入的区间不能使得原来的区间缩小，就会出现死循环，这个时候可以考虑将下取整：mid = (right - left) / 2 + left;

改变为上取整 mid = (right - left+1) / 2 + left;

在最后一次循环时， 如果向下取整，left可能会一直和mid相等，导致left值一直不变，进入死循环，所以，left不能赋值为mid; 如果向上取整，right可能会一直和mid相等，导致right值一直不变，进入死循环，所以，right不能赋值为mid;

public int guessNumber(int n) {
        int left = 1, right = n;
        while (left < right) { // 循环直至区间左右端点相同
            int mid = left + (right - left) / 2; // 防止计算时溢出
            if (guess(mid) <= 0) {
                right = mid - 1; // 答案在区间 [left, mid] 中
            } else {
                left = mid + 1; // 答案在区间 [mid+1, right] 中
            }
        }
        // 此时有 left == right，区间缩为一个点，即为答案
        return left;
    }
