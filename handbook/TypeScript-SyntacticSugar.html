    # 接口interface

    接口通常用来针对原型对象对象的遍历，用interface来定义约束对象的一个方式

    interface 主要用于定义类型（Type），而不是用于创建实例（构造函数）。它更多地用于类型检查和代码提示，以帮助开发者编写更加健壮的代码。

    interface Obj{
        name:string,
        age?:number//祖传的可选方式
}
const obj1:Obj = {
    name:'a',
    age:18
    //age:'18'//就会提示错误
}
const obj2:Obj = {
    name:'a'
}

    # 泛型（重要且常用）

    function myFn(a:number, b:number):number[]{
    return [a,b]
}
//如果想用它传其他类型的数据，就需要采用泛型了
////写法：在函数名的后面，参数的前面，写一个尖括号，里面写范型变量名
function myFn1<T>(a:T,b:T):T[]{
    return [a,b]
}
myFn1<number>(1,2)
myFn1<string>('aa','zz')//当然了，使用的时候不加尖括号也行，因为ts会类型推断，自动判断数据类型
myFn1('aa','zz')
//myFn1('aa',11)//以第一次类型推断的结果为准

    # 对同一个函数的两种赋值

    同名函数可以根据参数的类型不同以达到不同操作，这个功能好没用啊

    //函数参数提示符,指定针对不同参数，输入输出类型，在调用函数的时候会提示
function hello(age:number):string
function hello(name:string):string
//但是实际上，有没有都一样，提示而已
function hello(value: string|number): string{
    if(typeof value === 'string'){
        return 'my name is: '+value
    }else if (typeof value === 'number'){
        return 'my age is: '+value
    }else{
        return 'illegal'
    }
}
console.log(hello('lelly'))
console.log(hello(20))
//调用函数的时候，编译器会提示：function hello(age: number): string (+1 overload)

    # 接口继承

    //interface其实就是定义了一个名为parent的新类型，并对里面的值做了约束
interface Parent{
    prop1:string
    prop2:number
}
//可以实现类型的继承,使用extends关键字
interface Child extends Parent{

}

const myObj:Child = {

}

    # 类的修饰符

    可选属性 属性默认值 私有属性 受保护属性 静态成员 只读
    aaa?:string bbb = 100 private protected static readonly
    class Article{
    //设置属性
    title:string
    content:string
    aaa?:string//可选属性
    bbb = 100//属性默认值（用ts类型推断）
    public x = 123 //public,属性可以从外部调用
    private y = 456 //private,只能在类里面访问，外面获取不到（子类也不行）
    protected z = 789 //受保护的，只能在类中以及其子类里面使用，外界获取不到
    static author = 'Mio_Huang'//静态属性，只将属性设置给类本身，而不是设置给类的实例
    private static kkk = 1 //类修饰符可以嵌套（static只能放在最后）

    //初始化构造函数
    constructor(title:string,content:string){
        this.title = title//Property 'title' does not exist on type 'Article'.
        this.content = content//如果不设置初始属性，会报错：类型article上不存在属性content
    }
}
const a = new Article('标题','内容')
console.log(a.author)//输出public、private、protected 都可以，唯独输出static是undefined

class B extends Article{
    //继承也好，要在构造函数体上标上形参的呀
    constructor(title:string,content:string){
        super(title,content)
        this.y//private属性也不会继承
        this.author//看，static不会在在类的实例中
    }
}

    # 存取器

    存，会触发get方法，取，会触发set方法

    class User{
    private _password: string = ''

    get password(): string{
        return '*******'//读取时，只会获得这个值
    }

    set password(newPass:string){
        this._password = newPass
    }
}

const u = new User()
console.log(u.password)//读，触发get
u.password = '123456'//取，触发set

    # 抽象类

    抽象类里面可以有抽象的属性，方法，存取器。也可以有非抽象的属性

    abstract class Animal{
    //name:string = '123' //不抽象的类
    abstract name:string //抽象类不需要被实例化或者赋值
    abstract maskSound(): void //抽象方法不需要写内容（离谱）
    callMe():void{
        console.log('Mio_huang')
    }
}
class Cat extends Animal{
    //需要注意的是，在实例化抽象类时，抽象对象缺一不可，缺一个就会报错
    //因为在子类继承的时候，如果不给抽象类的抽象方法赋值内容，他就是空空如也
    name: string = '小猫'
    maskSound():void{
        console.log('Miao~')
    }
}

    # 类实现接口

    通过interface来定义类应该具有的行为：使用implements关键字

    好处是implements可以后面跟多个interface，用逗号隔开。优点是一个对象不可继承自多个类，但是这个可以补齐这个遗憾

    interface Animal{
    name:string
    get sound():string
    makeSound():void
}
//使用场景：用interface给类提供一个约束条件（）
class Dog implements Animal{
    //值得注意，这个类必须声明上述interface的全部属性，否则会报错
    name:string = 'dog'
    get sound(){
        return '狗叫'
    }
    makeSound():void{

    }
}

    # 类与泛型结合

    使用的时候发现，myclass要处理的数据，可能是一个string，也有可能是一个number，这时候就要用到泛型

    • 在类名的后面，大括号的前面加上, 然后就可以将T替代类里面的具体数据类型
    class MyClass<T>{
    value:T
    constructor (value:T){
        this.value = value
    }
    do():void{
        console.log(this.value)
    }
}
const myStr = new MyClass<string> ('hello')
const myNumber = new MyClass<number> (123321)
